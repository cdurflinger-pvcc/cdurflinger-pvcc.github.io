GitHub Q & A

  • Q. What is a version control system?
  • A. the practice of tracking and managing changes to software code
  • Q. What is Git?
  • A.a free and open source distributed version control system
  • Q. What is GitHub?
  • A. a website and cloud-based service that helps developers store and manage their code
  • Q. What is the difference between Git and GitHub?
  • A.Git is a version control system that lets you manage and keep track of you source code history and GitHub is a cloud-based hosting service that lets you manage Git repositories
  • Q. Who started GitHub and how was it started?
  • A. Tom Preston-Werner, Scott Chacon, Chris Wanstrath, and P.J. Hyett. To continue development of the Linux kernel
  • Q. What company owns it now?
  • A. Microsoft
  • Q.How much does a GitHub account cost?
  • A. Free
  • Q. What is the Octocat?
  • A. Mascot for GitHub

Git/GitHub terms:

  • Repository:
  • a central location in which data is stored and managed
  • Commit:
  • snapshot of your repository at a specific point in time
  • Fork:
  • take the source code from an open source software program and develop and entirely new program
  • Push:
  • a function that adds one or more elements to the end of an array
  • Pull requests:
  • an event that takes place in software development when a contributor/developer is ready to begin
  • Workflows:
  • sequence of industrial, administrative, or other processes through which a piece of work passes from initiation to completion
  • Issues:
  • any situation that occurs that is unexpected or prevents something from occurring
  • Raw button:
  • opens the file in a raw form, meaning that any HTML formatting disappears
  • Blame button:
  • a Git feature, designed to help you determine who made changes to a file
