Terms Definiiton
Software engineering the branch of computer science that seeks principles to guide the development of large, complex systems.
Integrated development environment (IDE) a software application that provides comprehensive facilities to computer programmers for software development.
Purpose of ACM a global scientific and educational organization dedicated to advancing the art, science, engineering, and application of computing,
Benefits of ACM membership provides a vast network to pros and students, along with targeted job opportunities.
Student ACM membership at $19 it allows access to online books and courses.
Purpose of IEEE nurtures, develops, and advances the building of global technologies.
Benefits of IEEE membership reduced rates on many proprietary products and services including IEEE books and eBooks, journals and articles, conferences and proceedings, standards, society memberships, and continuing education courses.
Student IEEE membership at $32 a year, you'll be given access to changes in tech, resources, and networking with pros
Requirements analysis start of the software life cycle; specify what services the proposed system will provide.
Design involves creating a plan for the construction of the proposed system.
Implementation involves the actual writing of programs,creation of data files, and development of databases.
Testing debugging programs and confirming that the final software product was compatible with the software development process.
Modular implementation a software design technique that emphasizes separating the functionality of a program into independent, interchangeable modules.
Unified Modeling Language (UML) a general-purpose, developmental modeling language in the field of software engineering that is intended to provide a standard way to visualize the design of a system.
Use case a list of actions or event steps typically defining the interactions between a role and a system to achieve a goal.
Use case diagram depicts the proposed system as a large rectangle in which interaction between the system and its users afe represented as ovals and users of the system are represented as stick figures.
One-to-one the association between patients and occupied provate rooms in that each patient is associated with only one room and each room is accosciated with one patient.
One-to-many the association between physicians and patients in that one physicians is assocoated with many patients and each patient is associated with one physician.
Many-to-many would occur if we included consulting physicians in the physician-patient relationship.
Design pattern a predeveloped model for sovling a recurring problem in software design.
Software quality assurance (SQA) charged with overseeing and enforcing the quality contol systems adopted by the organization.
Glass box testing software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
Black box testing test that do not rely on knowledge ofthe software's interior composition.
Beta testing a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software preforms in real-lifesituations before the final version of the product id solidified and released to the market.
Alpha testing the initial phase of validating whether a new product will perform as expected.
User documentation explain the features of the software and describe how to use them.
System documentation describe the software's internal composition so that the software can be maintained later in its life cycle.
Technical documentation descide how a software systme should be installed and serviced.
Software license a legal agreement between the owner and user of a software product that grants the user certain permissions to use the product with out transferring ownership rights to the intellectual property.
Software disclaimer the software developer's formal notice to the user of its software that the developer is denying any and all liability for any damages arising out of using the software as well as denying any implied warranties.
